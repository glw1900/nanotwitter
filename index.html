Technology Description

We adopt redis to cache data that requires comparatively high cost in querying in database and yet are requested frequently, such as 50 most recent twitters and timeline of recent login users.

Read into comments table for every twitter is quite a cost. We noticed that most twitters might not have comments, therefore there is no need to query in the comments table every time. Then we labeled every tweet as 1 or 0, when tweets are created, the label is set to 0, then if a comment is made, we change the label to 1. Thus every time we first check the label of a twitter, if itâ€™s labeled 0 then we check in the comment table.
